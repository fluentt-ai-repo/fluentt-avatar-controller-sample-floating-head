# v0.2.0 - Talking Animation 시스템 및 Energy Matching

**릴리스 날짜**: 2026-02-26

## 주요 변경사항

### 1. Talking Animation 시스템 (Phase 1)

Idle과 동일한 swap-buffer 패턴으로 대화 중 바디 애니메이션을 자동 교체하는 시스템을 추가했습니다.

- **가중치 기반 랜덤 선택**: `IdleAnimationEntry` 재사용, 각 클립별 weight/preventRepeat 지원
- **Idle ↔ Talking 전환**: `isTalking` Animator 파라미터로 상태 전환 (0.3초 CrossFade)
- **ExitTime 교차 재생**: Talking 0 ↔ Talking 1 상태 간 ExitTime=1.0 전환
- **OnSentenceStarted/OnLastSentenceStarted**: 음성 시작/종료 시 자동으로 SetTalking(true/false) 호출
- Talking 클립 미할당 시 Idle 상태 유지 (하위 호환)

### 2. Energy Matching 시스템 (Phase 2)

오디오 RMS 커브와 모션 에너지 커브를 비교하여 가장 유사한 Talking 클립을 자동 선택합니다.

- **모션 에너지 사전 계산**: Start() 시점에 각 Talking 클립을 `SampleAnimation()`으로 N개 시점 샘플링하여 bone position 변화량 기반 에너지 커브 생성 및 정규화
- **오디오 RMS 계산**: 문장 시작 시 `AudioClip.GetData()`로 PCM 캐싱 후 윈도우별 RMS 커브 생성
- **잔여 오디오 매칭**: swap 시점마다 `RefreshRemainingAudioRMS()`로 남은 오디오 구간의 RMS를 재계산하여 후속 클립 선택 정밀도 향상
- **Cosine Similarity**: 오디오 RMS vs 모션 에너지 커브 비교
- **최종 스코어**: `(1 - blendRatio) × normalizedWeight + blendRatio × similarity`
- **런타임 토글**: Inspector에서 `enableEnergyMatching` 체크박스로 실시간 ON/OFF 가능 (OFF→ON 시 lazy initialization)

#### 설정

| 필드 | 설명 | 기본값 |
|---|---|---|
| `enableEnergyMatching` | Energy Matching 활성화 토글 | false |
| `energyBlendRatio` | 0=가중치만, 1=유사도만 | 0.7 |

#### Fallback 조건

- Talking 클립 1개 → weighted random
- AudioClip null (NoServer 모드) → weighted random
- 잔여 오디오 0.5초 미만 → weighted random
- enableEnergyMatching = false → weighted random

### 3. SwapBufferNotifier 아키텍처 개선

기존 Update 기반 transition 감지를 StateMachineBehaviour 방식으로 교체하여 swap 타이밍 문제를 해결했습니다.

#### 기존 방식 (제거)
```
Update() → CheckIdleSwap() / CheckTalkingSwap()
  → wasInTransition && !isInTransition 감지
  → transition 완료 직후 swap → 끊김 발생
```

#### 새 방식
```
SwapBufferNotifier (StateMachineBehaviour)
  → OnStateEnter: pendingSwap = true
  → OnStateUpdate: !IsInTransition 확인 후 OnSwapSlotReady() 호출
  → 상태 완전히 settle된 후 dormant 슬롯 swap → 끊김 없음
```

- **BufferGroup enum**: Idle / Talking 구분
- **slotIndex**: 0 또는 1
- **에디터 자동 설정**: "Setup SwapBuffer Behaviours" 버튼으로 Idle 0/1, Talking 0/1 상태에 자동 부착

### 4. SampleAnimation 안전 처리

모션 에너지 사전 계산 시 `SampleAnimation()`이 실제 Transform을 수정하는 부작용 방지:

- bone Transform (localPosition, localRotation, localScale) 저장/복원
- BlendShape 값 저장/복원 (head_skmr 리스트 활용)
- Start() 첫 프레임에서 실행 → 렌더링 전 완료
