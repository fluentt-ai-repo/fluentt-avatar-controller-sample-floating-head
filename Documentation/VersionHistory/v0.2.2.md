# v0.2.2 - 코드 품질 개선 (파일 분할, 네이밍, 매직 넘버 상수화)

**릴리스 날짜**: 2026-03-02

## 주요 변경사항

### 1. 대형 파일 분할

#### LookTargetController.cs (800줄 → 4개 파일)
- `LookTargetTypes.cs` — enum/class 타입 정의 분리 (ELookTargetStrategy, LookTargetSetting, EEyeControlStrategy, EyeBlendShape, EyeBlendShapes)
- `LookTargetController.cs` — partial class 핵심 컨트롤러 + head tracking
- `LookTargetController.EyeTransform.cs` — Eye Transform/Corrected 모드 업데이트
- `LookTargetController.EyeBlendShape.cs` — BlendShape 초기화, 계산, 적용 + 매직 넘버 상수화

#### Editor LookTarget (939줄 → 3개 파일)
- `FluentTAvatarControllerFloatingHeadEditor.LookTarget.cs` — Inspector UI 드로잉 (DrawLookTargetSettings만)
- `FluentTAvatarControllerFloatingHeadEditor.LookTargetRigSetup.cs` — Rig 자동 구성/해체 + virtual target 생성
- `FluentTAvatarControllerFloatingHeadEditor.LookTargetToggle.cs` — Head/Eye tracking 토글 + BlendShape 자동 검색

#### FluentTAvatarControllerFloatingHead.LookTarget.cs → 2개 파일
- `FluentTAvatarControllerFloatingHead.LookTarget.cs` — 초기화 + 제어/업데이트 로직
- `FluentTAvatarControllerFloatingHead.LookTargetGizmo.cs` — OnDrawGizmos 시각화 코드 분리

#### 데이터 타입 분리
- `AvatarControllerFloatingHeadTypes.cs` — IdleAnimationEntry, GestureMapping을 메인 컨트롤러에서 분리

### 2. 네이밍 수정

| 변경 전 | 변경 후 | 이유 |
|---------|---------|------|
| `head_skmr` | `headSkinnedMeshRenderers` | snake_case 위반 + 약어 모호 |
| `skmr` (EyeBlendShape 필드) | `skinnedMeshRenderer` | 약어 풀어쓰기 |
| `isLookingEye` | `isEyeTracking` | 모호한 표현 명확화 |
| `isFadingOutEye` | `isEyeFadingOut` | 주어-동사 순서 통일 |
| `quadLeftEyeBallFromHead` | `rotLeftEyeFromHead` | Quaternion rotation 의미 명확화 |
| `quadRightEyeBallFromHead` | `rotRightEyeFromHead` | 동일 |

- `head_skmr`, `skmr` 필드에 `[FormerlySerializedAs]` 적용하여 기존 직렬화 데이터 호환성 유지
