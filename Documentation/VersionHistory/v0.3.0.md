# v0.3.0 - 자료구조 통일, TalkMotion Idle, overrideEyeBlink, Drag & Drop

**릴리스 날짜**: 2026-04-02

## ⚠️ Breaking Changes

- **모든 애니메이션 엔트리 타입이 `AnimationEntryBase` 기반으로 통일**
- Inspector에 입력된 기존 애니메이션 데이터가 초기화됩니다. 재입력 필요.
- `GestureMapping.animationClips` (List\<AnimationClip\>) → `clips` (List\<AnimationEntryBase\>)
- `GestureMapping.blendWeight`, `overrideEyeControl` 제거 (각 clip별로 이동)
- `OneShotMotionEntry.blendWeight` 제거 (`AnimationEntryBase.weight`로 대체)

## 주요 변경사항

### 1. 자료구조 통일 (`AnimationEntryBase`)

공통 base class 도입으로 모든 애니메이션 엔트리가 동일한 필드 구조를 공유:

```
AnimationEntryBase: clip, weight, overrideEyeControl, overrideEyeBlink
├─ IdleAnimationEntry: + preventRepeat
├─ OneShotMotionEntry: + motionId
├─ OneShotMotionGroupEntry: (base만)
└─ GestureMapping.clips: List<AnimationEntryBase>
```

### 2. overrideEyeBlink

모든 애니메이션 타입에 `overrideEyeBlink` 옵션 추가:
- 활성화 시 해당 클립 재생 중 Eye Blink 자동 일시 정지
- 클립에 eyeBlink 커브가 포함된 경우 자동 감지 (Inspector에서 클립 할당 시)
- `SuspendEyeBlink()` / `RestoreEyeBlinkIfSuspended()` API 추가

### 3. TalkMotion Idle 전환 (idleSwap trigger)

- `talkMotionIdleClip` — 대화 중 재생할 기본표정 idle 클립
- 대화 시작/종료 시 `idleSwap` trigger로 비활성 슬롯에 클립 로드 후 Animator 전환
- 전환 중(IsInTransition) 안전 대기 — 양쪽 슬롯 사용 중일 때 전환 완료 후 실행
- Animator Controller에 `idleSwap` trigger + 전환 자동 생성 (Setup SwapBuffer Behaviours 버튼)

### 4. Drag & Drop 지원

Inspector에서 AnimationClip을 드래그 앤 드롭으로 추가 가능:
- **Idle Animations**: 클립 드롭 → IdleAnimationEntry 자동 생성 (weight=1)
- **One-Shot Motions**: 클립 드롭 → OneShotMotionEntry 자동 생성 (motionId=클립이름)
- **Gesture Mapping clips**: 클립 드롭 → AnimationEntryBase 자동 생성
- 모든 드롭에서 overrideEyeControl/overrideEyeBlink 자동 감지

### 5. GestureMapping weight 기반 선택

- 기존 uniform 랜덤 → weight 기반 랜덤 선택으로 변경
- 각 클립별 개별 weight 설정 가능

### 6. Idle Animation overrideEyeControl

- `IdleAnimationEntry`에 `overrideEyeControl` 추가
- 슬롯 전환 시 활성 idle의 설정에 따라 Eye Control 자동 suspend/restore
