# v0.1.8 - 멀티 Idle 애니메이션 시스템 및 override 클립 네이밍 정리

**릴리스 날짜**: 2026-02-24

## 주요 변경사항

### 1. 멀티 Idle 애니메이션 시스템

기존 단일 `defaultIdleAnimationClip`을 여러 Idle 클립을 가중치 기반 랜덤으로 교체 재생하는 시스템으로 확장했습니다.

- **가중치 기반 랜덤 선택**: `IdleAnimationEntry`에 `weight` 필드로 각 클립의 선택 확률 조절
- **swap-buffer 패턴**: Gesture 레이어와 동일한 양 슬롯 교차 방식으로 자연스러운 전환
- **preventRepeat**: 같은 클립이 연속 재생되지 않도록 방지하는 옵션
- **Animator ExitTime 전환**: Base 레이어에 Idle 0 ↔ Idle 1 양방향 전환 추가 (ExitTime=1.0, 0.3초 블렌드)

#### 엣지 케이스 처리

| 상황 | 동작 |
|---|---|
| 클립 1개 | 양쪽 슬롯에 동일 클립 override → 단일 루프 재생 |
| 클립 1개 + preventRepeat | 경고 로그, preventRepeat 무시 |
| 모든 가중치 0 | 경고 로그, 균등 확률로 선택 |
| 클립 0개 | override 없이 기본 placeholder 클립 사용 |

#### 레거시 마이그레이션

기존 `defaultIdleAnimationClip` 필드가 설정된 프리팹은 런타임 시 자동으로 `idleAnimations` 리스트로 마이그레이션됩니다.

### 2. Override 클립 네이밍 정리

`*_dummy` 네이밍을 역할이 명확한 `*_override` / `*_idle` 컨벤션으로 통일했습니다.

| 기존 | 변경 후 | 역할 |
|---|---|---|
| `default_dummy` | `idle_override_0` | Base 레이어 Idle 슬롯 0 |
| *(신규)* | `idle_override_1` | Base 레이어 Idle 슬롯 1 |
| `emotion_dummy_0` | `gesture_override_0` | Gesture 레이어 슬롯 0 |
| `emotion_dummy_1` | `gesture_override_1` | Gesture 레이어 슬롯 1 |
| `default_emotion_dummy` | `gesture_idle` | Gesture 레이어 Idle 상태 (비 override) |

### 3. CustomEditor [Header] 어트리뷰트 중복 제거

`PropertyField`가 `[Header]` 데코레이터를 자동 렌더링하여 커스텀 에디터의 `LabelField`와 중복 표시되는 문제를 수정했습니다. 런타임 클래스의 `[Header]` 어트리뷰트 10개를 제거하고 일반 주석으로 대체했습니다.
