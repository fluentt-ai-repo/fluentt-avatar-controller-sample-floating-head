# v0.2.5 - EyeBlink BlendMode 기본값 변경 및 런타임 API 추가

**릴리스 날짜**: 2026-04-01

## 주요 변경사항

### 1. blinkBlendMode 기본값 변경: Override → SoftMax2D

TalkMotion 재생 중 eyeBlink 값이 높은 상태(눈 감김)에서 눈 깜빡임 애니메이션이 트리거될 때 발생하던 지글거림(jittering) 현상을 해결합니다.

- **원인**: Override 모드에서 face 레이어의 eyeBlink 값(~100)을 blink 레이어가 Lerp로 대체하면서 0→100 구간에서 빠른 왕복 발생
- **해결**: SoftMax2D 모드는 두 값 중 큰 값을 부드럽게 취하므로, 이미 눈이 감긴 상태에서 blink이 트리거되어도 시각적 이상 없음

### 2. SetBlinkBlendMode() 런타임 API 추가

런타임에서 `blinkBlendMode`를 변경할 때 즉시 blink 레이어에 반영되도록 `SetBlinkBlendMode()` 메서드를 추가했습니다.

```csharp
// 런타임에서 블렌드 모드 즉시 변경
controller.SetBlinkBlendMode(TMAnimationLayer.BlendMode.Override);
```

기존에는 `InitializeEyeBlink()` 호출 시점(Start)에만 레이어에 적용되어, 런타임 변경이 반영되지 않았습니다.

## 변경 파일

### Runtime
- `FluentTAvatarControllerFloatingHead.cs` — blinkBlendMode 기본값 Override → SoftMax2D
- `FluentTAvatarControllerFloatingHead.EyeBlink.cs` — SetBlinkBlendMode() 메서드 추가

## 하위 호환성

✅ **Breaking Change 없음**: 기존 프로젝트는 직렬화된 Override 값이 유지됩니다. 새 프로젝트에만 SoftMax2D 기본값이 적용됩니다. SetBlinkBlendMode()는 비파괴적 API 추가입니다.
