# v0.1.6 - 클라이언트 감정 태깅 빌드 타임 전환 및 멀티 클립 지원

**릴리스 날짜**: 2026-02-23

## 주요 변경사항

### 1. 클라이언트 감정 태깅: 빌드 타임 트리거로 전환

기존에는 재생 시점(`onSentenceStarted`)에서 감정 태깅을 처리했으나, 이제 타임라인 빌드 시점(`onSubtitleContentAdded`)에서 처리합니다.

**변경 이유**:
- 기존 방식은 `emotionTaggedData` HashSet으로 중복 처리를 방지했으나, Prepare 탭 캐시 재생 시 동일 인스턴스가 차단되는 버그 발생
- 빌드 타임 처리로 전환하여 HashSet 없이 매번 정확하게 마커 추가

**동작**:
- `onSubtitleContentAdded` 이벤트 구독 → 자막이 타임라인에 추가될 때 regex 감정 분석 수행
- `AddMotionTagMarkerAbsolute()` API로 절대 타임라인 시간에 마커 배치
- batch edit 블록 안에서 실행되므로 추가 `RebuildGraph` 비용 없음

### 2. EmotionKeywordDataset 기반 Regex 감정 감지

ScriptableObject 기반 `EmotionKeywordDataset`에서 감정 키워드를 관리합니다.

**구성**:
- 감정별 regex 패턴 및 가중치 정의
- 초기화 시 감정별로 패턴을 그룹핑하여 단일 compiled regex로 결합
- 가중치 기반 확률적 후보 선택 (`maxEmotionTagsPerSentence`로 최대 개수 제한)

### 3. EmotionMotionMapping 멀티 클립 지원

감정 태그당 여러 AnimationClip을 지정하여 랜덤 변형 선택이 가능합니다.

**변경**:
- `AnimationClip animationClip` → `List<AnimationClip> animationClips`
- 재생 시 `Random.Range(0, count)`로 랜덤 클립 선택
- 클립을 1개만 지정하면 기존과 동일하게 동작

### 4. durationOverride 필드 제거

`EmotionMotionMapping`에서 사용되지 않던 `durationOverride` 필드를 제거했습니다.
